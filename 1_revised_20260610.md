
## 1. 现在最应该先做的 oracle 诊断

在微调之前，建议先跑四个 diagnostic upper bound。

### 诊断 1：SAM candidate recall

FSC147 有 dot annotations，所以可以检查每个 GT dot 是否被至少一个 SAM candidate 覆盖。

```python
dot_recall = num_dots_covered_by_any_candidate / num_gt_dots
```

按 GT 区间统计：

```text
0–10
11–20
21–50
51–100
100+
```

如果 51–100 的 dot recall 已经低，那分类头再训也救不了。

---

### 诊断 2：oracle category

把所有被 dot 匹配到的 candidate 类别强行改成 GT target category，然后跑后面的 clustering + dedup + counting。

如果 oracle category 后 MAE 大幅下降：

```text
分类头是主瓶颈
```

如果下降不多：

```text
relation / dedup / proposal 才是主瓶颈
```

---

### 诊断 3：oracle dedup

用 dot id 做 same-instance component。

规则：

```text
包含同一个 dot 的 candidates -> 同一 instance
包含不同 dot 的 candidates -> 不同 instance
```

然后看 count 能到什么程度。

如果 oracle dedup 明显好于原来的 A_inst：

```text
A_inst 阈值或关系头有问题
```

---

### 诊断 4：candidate upper bound

直接数：

```text
有多少 GT dot 至少能找到一个合理 candidate
```

这个就是当前 SAM2 proposal + filtering 的理论上限。

如果这个上限低，比如只有 60%，那么现在的 NAE 0.48 很正常。

---

## 2. 当前的 SAM 最大候选 128 需要调整

你说 100 以上可以忽略，这对目前分析可以。但 51–100 已经明显变差，说明 128 个 candidate 也可能不够。

原因是一个真实实例通常不只对应一个 SAM candidate：

```text
完整 mask
局部 mask
重复 mask
合并 mask
背景 mask
```

如果 GT=80，而每个实例平均需要 2 个候选才能保证覆盖，那 128 已经紧了。

建议至少试：

```text
max_candidates = 256
max_candidates = 512
```

然后看：

```text
dot recall 是否上升
duplicate rate 是否上升
最终 MAE 是否下降
runtime 是否还能接受
```

对于 FSC147 这种高密度 counting 数据，后续更推荐：

```text
multi-scale tiling + local SAM candidates + tile boundary merge
```

而不是单图全局最多 128 个候选。

## 3. 下一步优先级

我建议按这个顺序搞。

### Step 1：先做诊断，不训练

先跑：

```text
candidate dot recall
oracle category
oracle dedup
candidate upper bound
signed error: pred - gt
```

尤其要加 signed error。现在只有 MAE，不知道你是系统性 under-count 还是 over-count。

---

### Step 2：提高 candidate 上限

先试：

```text
128 -> 256 -> 512
```

如果 dot recall 明显上升，说明你现在高 count 区间主要是 proposal cap 问题。

---

### Step 3：用 FSC147 exemplar boxes 微调分类头

先只训干净 box，不挖 pseudo candidate。

训练策略：

```text
freeze DINO / SAM
train classification head or projection head only
LR 小一点
混入 LVIS replay
只训练 FSC category 能映射到 LVIS 的样本
```

如果是 text prototype head，可以把 FSC147 train category names 加到 vocab 里，训练 projection。

---

### Step 4：再加 dot-mined pseudo candidates

等 Step 3 有提升后，再用 dots 挖高置信 SAM candidates。

正样本只用：

```text
包含 exactly one target dot
面积合理
和 exemplar 相似
SAM score 高
```

不要把无 dot candidates 当强负样本。

---

### Step 5：关系头继续以实例分割数据为主

FSC147 没有完整 mask，所以不要让它主导 relation head。

可以后续做弱监督 fine-tune：

```text
same-instance positive: candidates containing same dot
same-instance negative: candidates containing different dots
part-whole weak label: same dot + mask containment
```

但 loss 权重要低，作为 adaptation，不作为主训练。

---

## 4. 最直接的实验矩阵

下一轮可以跑这个小矩阵：

| 实验 | 分类头                                    | 关系头          | SAM候选   | 目的                      |
| -- | -------------------------------------- | ------------ | ------- | ----------------------- |
| E0 | LVIS                                   | 当前           | 128     | 当前 baseline             |
| E1 | LVIS                                   | 当前           | 256/512 | 看 proposal cap          |
| E2 | oracle category                        | 当前           | 256     | 看分类瓶颈                   |
| E3 | LVIS + FSC exemplar 微调                 | 当前           | 256     | 看 FSC domain adaptation |
| E4 | LVIS + FSC exemplar + pseudo candidate | 当前           | 256     | 看弱监督候选训练                |
| E5 | E3 分类头                                 | oracle dedup | 256     | 看 relation/dedup 上限     |

如果 E2 提升最大，优先训分类头。
如果 E1 提升最大，优先调 SAM candidate。
如果 E5 提升最大，优先调 A_inst / representative selection。

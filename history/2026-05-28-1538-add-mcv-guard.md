summary:
- MCV v1 (`pred_strategy=mode_cluster_vote`) was empirically validated on
  FSC-147 val 1/3 in run `results/2026-05-28-1434-eval-mp7-mcv-full/` and
  found to **regress on 1-10 / 11-50 buckets** (MAE 6.5 → 16.7 / 7.4 → 14.9)
  while improving 51-200 / 201+ (35.2 → 25.1 / 313.0 → 280.7). Net: overall
  MAE 30.71 → 32.00 (+1.29), RMSE 93.9 → 90.0 (-4.0).
- Failure mode F1 (mode contamination, predicted in `MCV-method.md` §6) is
  the cause: on small-count images the largest non-singleton cluster is
  often a same-scale background fragment with size ≈ 5-15; MAD-based mode
  membership then promotes neighbouring background clusters into the
  mode-set and the count is over-estimated.
- This change adds an explicit `mcv_min_anchor_size` threshold: when the
  anchor (largest non-singleton) cluster has fewer members than this
  threshold, MCV is bypassed and `max` is used. Threshold sweep on the
  v1 trace (using saved per-image `trace.cluster_sizes`) shows a stable
  plateau at A ∈ [30, 40] with overall MAE ≈ 28.9-29.5, vs MP7+max
  baseline 30.71. We pick A=30 (start of the plateau, simpler integer,
  uses MCV on ~41% of images).
- Honest accounting: MCV originally claimed "zero new hyperparameter".
  Adding `mcv_min_anchor_size` violates that claim. The new flag defaults
  to `0` (no guard, original v1 behaviour), so the claim still holds when
  the flag is unset. The improved configuration is reported as a separate
  variant.

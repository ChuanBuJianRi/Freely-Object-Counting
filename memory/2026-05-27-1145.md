key_decisions:
- **GPU guard module is shared, not per-script.** Pulled `_gpu_temp` / `_throttle_if_hot` out of `eval_fsc147_full.py` into `codes/eval/_gpu_safety.py` so future evaluators (CARPK, OmniCount, ablation harnesses) all wire the same `GpuGuard.add_cli_args(parser)` + `guard_from_args(args)` and emit the same `metrics.json::thermal` block. Auto-detects `nvidia-smi` across PATH and four absolute fallbacks; gracefully no-ops on CPU-only machines.
- **`metrics.json::thermal` block is mandatory** for every GPU run (per AGENTREAD §6 + `results/index.md::GPU_THERMAL_POLICY`). Contains `peak_temp_c`, `cooldown_events`, `cooldown_seconds`, `polls`, plus the four config knobs. CPU-only runs include the block too, but with `enabled=false` and a `reason` string.
- **CPU-only runs are an explicit, documented exception.** Not a "loophole." A CPU-only run still gets a folder under `results/<run_id>/`, a README, and a `thermal:` block in `metrics.json` — the difference is only that `enabled=false` plus a one-line `reason`.
- **`sng_cluster` delta default flipped from `3` to `None` (= adaptive).** Backward-compatible because all 21 historical SNG sub-runs pass an explicit integer δ; only fresh callers that omit δ get the new behaviour. `OccamConfig.sng_alpha = 0.4` is the production default (matches the FSC-147 manually-tuned sweet-spot η ≈ 0.4); the synthetic-validation finding that α=0.5 is 14 % better has been documented in `library/notes/SNG-method.md §7.1.x`, but I deliberately did not change the default — the synthetic α=0.5 win is on a single SNR point and must be confirmed on real FSC-147 features before promotion. Left a precise reproduction command in the run README + `history/2026-05-27-1150-add-adaptive-sng-delta.md` for the next agent.
- **`adaptive_delta` is clamped to `[0, ε−2]`.** §6.2 of SNG-method.md gives the strict upper bound `δ < ε − 1`; `floor(...)` could otherwise round up to ε−1 when α ≈ 1, defeating the bound and causing intra-class pruning collapse (see ε=5,δ=5 row). The clamp makes the formula robust by construction; tested via the synthetic validator's α=1 corner.
- **Recovered the upstream nested `occam_multi/results/` layout** for byte-aligned diff with FreeCounting. The flat copy under the run's top level was redundant and made `diff -rq` noisy.
